BusMaster function

Q:How many MByte can be set for data buffer used in sampling/generating at most?

A:The value depends on the system you use, the following result reference value is based on the state which the other application isn't run.

CPU:Pentium4 2.40GHz
Memory:512MByte
Distribution: RedHat Linux 9
Condition: Buffer can be normally set by DioDmSetBuff().
Result: 63MByte

Notice: In Red Hat Professional Workstation, it can set up only to 15MByte.

Q:How many can be set for sampling clock at most in infinite transfer?

A:The value depends on the system you use, the following result reference value is based on the state which the other application isn't run.

CPU:Pentium4 2.40GHz
Distribution: RedHat Linux 9
Condition: In input infinitely transfers, transfer can process which uses internal clock and does not occur "FifoFull" error.
Result: 13.3MHz

CPU:Pentium4 2.40GHz
Distribution: RedHat Linux 9
Condition: In output infinitely transfers, transfer can process which uses internal clock and does not occur "FifoEmpty" error.
Result: 6.6MHz

Q:In generating, the data is outputted with a delay of 10ms after doing DioDmStart.

A:In generating, by default, a Wait of 10ms would be invoked in order to wait for data to stand in Fifo. But, sometimes the wait may not be invoked if the output data are few. When Wait becomes a problem, please change the following item of the setting file "contec_dio.conf" under a config directory.

Modify "WaitTimeWhenOutput:Number:10" by 10ms unit and restart the driver.
